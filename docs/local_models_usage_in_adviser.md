# Выводы по локальным моделям (Ollama) и минимальная архитектура

## 1. Текущий набор моделей

Установленные модели:

- qwen2.5-coder:1.5b
- qwen2.5-coder:0.5b
- qwen2.5-coder:1.5b-base
- nomic-embed-text

---

## 2. Анализ моделей

### 2.1 qwen2.5-coder:1.5b

Сильные стороны:
- высокая скорость (подходит для CPU)
- хорошо работает с кодом
- стабильно генерирует JSON и структуры
- подходит для небольших функций

Слабые стороны:
- слабый reasoning
- плохо справляется со сложной логикой
- ограниченный контекст

Роль:
- основной fast code executor

---

### 2.2 qwen2.5-coder:0.5b

Сильные стороны:
- очень высокая скорость
- минимальная нагрузка на систему

Слабые стороны:
- низкое качество
- частые ошибки даже в простых задачах

Роль:
- вспомогательный ultra-fast слой
- простые преобразования

---

### 2.3 qwen2.5-coder:1.5b-base

Особенности:
- не является instruct моделью
- требует сложного prompt engineering

Рекомендация:
- не использовать

---

### 2.4 nomic-embed-text

Назначение:
- генерация эмбеддингов
- используется для поиска и RAG

Роль:
- не участвует в генерации
- пригодится позже

---

## 3. Главные выводы

1. Есть быстрый слой (0.5B / 1.5B)
2. Нет универсальной модели
3. Нет полноценного reasoning
4. Система не подходит для сложных задач без fallback

---

## 4. Минимальная архитектура (только локальные модели)

### Компоненты:

- Router (rules)
- Code Executor (1.5B)
- Fast Helper (0.5B)

---

## 5. Routing (максимальная эффективность)

```python
def route(task):
    if task["type"] == "code":
        if task["complexity"] == "low":
            return "qwen2.5-coder:0.5b"
        else:
            return "qwen2.5-coder:1.5b"

    if task["type"] == "transform":
        return "qwen2.5-coder:1.5b"

    return "qwen2.5-coder:1.5b"
```

---

## 6. Правила использования моделей

### Для 0.5B:
- только простые задачи
- без логики
- без длинных инструкций

Примеры:
- форматирование JSON
- простые шаблоны
- замена текста

---

### Для 1.5B:
- основной рабочий слой
- небольшие функции
- SQL
- API glue

Примеры:
- написать функцию
- преобразовать структуру
- собрать JSON

---

## 7. Ограничения системы

- нельзя делать сложные архитектуры
- нельзя делать multi-step reasoning
- нельзя полагаться на “мышление” моделей
