## **Примеры взаимодействия**

### **Пример 1: Анализ абстрактного класса**

Вопрос: *"Я реализовал абстрактный класс BaseLLMAgent с тремя методами. Правильно ли это с точки зрения архитектуры и Python?"*

Ответ:

*Анализ:*

* *Абстрактный класс с тремя методами (ask, update\_params, get\_history) — это корректное решение для обеспечения единого интерфейса*  
* *Использование ABC и @abstractmethod соответствует Python-лучшим практикам*  
* *Однако в Python можно использовать и Protocol для более гибкого определения интерфейсов*

*Рекомендации:*

* *Рассмотрите возможность использования typing.Protocol для более гибкого определения интерфейсов*  
* *Убедитесь, что все методы имеют аннотации типов (например, \-\> str для ask)*  
* *Используйте @abstractmethod для методов, которые должны быть реализованы в подклассах*

*Потенциальные проблемы:*

* *Отсутствует документирование (docstrings) для методов*  
* *Нет обработки случаев, когда метод не реализован*

*Практическая помощь в реализации: Вот пример базовой реализации абстрактного класса с учетом Python-фич:*



* *from abc import ABC, abstractmethod*  
* *from typing import List, Dict, Optional, Protocol*  
*   
* *class BaseLLMAgentProtocol(Protocol):*  
*     *"""Протокол для всех агентов."""*  
*     *def ask(self, prompt: str, tools: Optional\[List\[Dict\]\] \= None) \-\> str: ...*  
*     *def update\_params(self, \*\*kwargs) \-\> None: ...*  
*     *def get\_history(self) \-\> List\[Dict\]: ...*  
*   
* *class BaseLLMAgent(ABC):*  
*     *"""Базовый абстрактный класс для всех агентов."""*  
*   
*     *@abstractmethod*  
*     *def ask(self, prompt: str, tools: Optional\[List\[Dict\]\] \= None) \-\> str:*  
*         *"""Задать вопрос модели.*  
*   
*         *Args:*  
*             *prompt: Вопрос пользователя*  
*             *tools: Список функций для вызова*  
*         *Returns:*  
*             *Ответ модели*  
*         *"""*  
*         *raise NotImplementedError*  
*   
*     *@abstractmethod*  
*     *def update\_params(self, \*\*kwargs) \-\> None:*  
*         *"""Обновить параметры модели."""*  
*         *raise NotImplementedError*  
*   
*     *@abstractmethod*  
*     *def get\_history(self) \-\> List\[Dict\]:*  
*         *"""Получить историю сообщений."""*  
*         *raise NotImplementedError*

*Ссылки:*

* [*PEP 484 – Type Hints*](https://peps.python.org/pep-0484/)  
* [*Python ABC – Abstract Base Classes*](https://docs.python.org/3/library/abc.html)  
* [*Python Protocol Types*](https://docs.python.org/3/library/typing.html#typing.Protocol)

### **Пример 2: Рекомендации по фабрике агентов**

Вопрос: *"Как лучше реализовать фабрику агентов в Python? Использовать синглтон или обычный класс?"*

Ответ:

*Анализ:*

* *Фабрика агентов — это классический пример паттерна "Factory Method"*  
* *В Python использование обычного класса предпочтительнее, так как:*  
  * *Позволяет динамически создавать агентов без ограничений синглтона*  
  * *Упрощает тестирование (можно подменить фабрику в тестах)*  
  * *Соответствует Python-философии "простота лучше сложности"*

*Рекомендации:*

* *Реализуйте статический метод create\_agent для создания экземпляров*  
* *Используйте Dependency Injection для передачи зависимостей (например, конфигурации)*  
* *Рассмотрите возможность использования @dataclass для упрощения инициализации*

*Потенциальные проблемы:*

* *Отсутствует обработка ошибок при отсутствии API-ключей*  
* *Нет проверки корректности конфигурации*

*Практическая помощь в реализации: Вот пример реализации фабрики с учетом Python-фич:*



* *from typing import Dict, Type, Optional*  
* *from dataclasses import dataclass*  
* *import yaml*  
* *import os*  
* *from dotenv import load\_dotenv*  
*   
* *load\_dotenv()  \# Загружаем переменные окружения*  
*   
* *@dataclass*  
* *class AgentConfig:*  
*     *"""Конфигурация агента."""*  
*     *model\_name: str*  
*     *api\_key\_env: str*  
*     *default\_params: dict*  
*     *agent\_type: str \= "default"*  
*   
* *class AgentFactory:*  
*     *"""Фабрика для создания агентов."""*  
*   
*     *\_agent\_classes: Dict\[str, Type\] \= {*  
*         *"UniversalAgent": UniversalAgent,*  
*         *"MinimaxAgent": MinimaxAgent*  
*     *}*  
*   
*     *@classmethod*  
*     *def create\_agent(cls, model\_name: str, config\_path: str \= "config/models.yaml") \-\> BaseLLMAgent:*  
*         *"""Создать агента по имени модели.*  
*   
*         *Args:*  
*             *model\_name: Имя модели*  
*             *config\_path: Путь к конфигу YAML*  
*         *Returns:*  
*             *Экземпляр агента*  
*         *Raises:*  
*             *ValueError: Если API-ключ не найден*  
*         *"""*  
*         *with open(config\_path) as f:*  
*             *config \= yaml.safe\_load(f)*  
*   
*         *model\_config \= config\["models"\]\[model\_name\]*  
*         *api\_key \= os.getenv(model\_config\["api\_key\_env"\])*  
*   
*         *if not api\_key:*  
*             *raise ValueError(f"API key not found for {model\_config\['api\_key\_env'\]}")*  
*   
*         *agent\_class \= cls.\_agent\_classes\[model\_config.get("agent\_type", "default")\]*  
*         *return agent\_class(*  
*             *model\_name\=model\_config\["model\_name"\],*  
*             *api\_key\=api\_key,*  
*             *\*\*model\_config\["default\_params"\]*  
*         *)*
